## Regressões checadas

- **Middleware existente:** outras rotas `(dashboard)/**` agora exigem sessão. Rotas `(auth)/**` ficam públicas, exceto `/login/redefinir-senha` (que precisa ser acessível tanto logado quanto não). Dashboard antigo (`/cells`, `/members`, `/meetings`) permanece funcionando — só ganhou gate.
- **Header hardcoded "Admin Silva":** removido. Header agora recebe prop `user` obrigatória. Qualquer lugar que renderizava Header sem prop quebraria — verificado: só é renderizado pelo `layout.tsx` do `(dashboard)`.
- **Sidebar:** `role` é opcional para não quebrar sheet mobile do Header, que passa o role explicitamente.
- **Typecheck:** `npx tsc --noEmit` limpo em `src/`. Os 38 erros remanescentes estão todos em `dashboard/src/` (projeto Opensquad Office, pré-existente, fora do escopo).

## Observações / follow-ups (não bloqueiam)

### Observação 1 — Igreja não é selecionável no form de novo usuário
**Severidade:** baixa
**Descrição:** o critério 4.2 lista "igreja" como campo do form. O código injeta automaticamente o `churchId` do ADMIN logado (`createUser` em `users.ts:138`). Isso é coerente com o modelo multi-tenant (cada ADMIN só enxerga sua própria igreja) e foi endossado pelo plano técnico. A UX ficou correta: sem campo desnecessário. Registro aqui apenas para o PM confirmar que a simplificação é desejada quando evoluirmos para múltiplas igrejas por ADMIN.
**Ação sugerida:** nenhuma agora. Reabrir quando US-06 "trocar de igreja" existir.

### Observação 2 — Verificação manual do envio real de email pelo Resend
**Severidade:** média (depende de config)
**Descrição:** o código chama `supabase.auth.resetPasswordForEmail(..., { redirectTo })`. O envio real depende de o SMTP do Resend estar configurado no painel do Supabase Auth e da env `NEXT_PUBLIC_SITE_URL` apontar para a URL correta. QA estático não cobre isso.
**Ação sugerida:** Patrick valida manualmente após subir as envs (ver "Testes manuais" abaixo).

### Observação 3 — Campo `phone` em `editar` não está nos critérios explícitos
**Severidade:** baixa
**Descrição:** o PM listou "alterar nome, papel, supervisor" para edição. O form de edição também permite editar `phone`. Não está no critério, mas não contradiz nada e melhora a ergonomia. Registro para conferência.
**Ação sugerida:** nenhuma — assumir como ganho de escopo minor.

### Observação 4 — `src/app/(dashboard)/settings/page.tsx` placeholder
**Severidade:** baixa
**Descrição:** o caminho `/settings` (sem `/usuarios`) continua sendo o placeholder antigo. ADMIN é levado direto para `/settings/usuarios` pela Sidebar, mas um deep link em `/settings` mostra placeholder vazio.
**Ação sugerida:** no próximo sprint, transformar `/settings` em menu lateral de configurações quando houver mais de uma seção.
